## 1. Eval set

`evaluation_set.json` at the repo root. 20 items, brief's exact
distribution:

| Category   | Count | Notes                                                                  |
|------------|------:|------------------------------------------------------------------------|
| factual    |     8 | Single-doc lookups; substring containment + recall@k.                  |
| multi_hop  |     5 | 2+ docs must surface in the citations.                                 |
| conflict   |     3 | Both contradicting docs must be surfaced; system flags supersession.   |
| bahasa     |     2 | Bahasa Indonesia Q + A; per-locale rubric.                             |
| trick      |     2 | Should respond "I don't have that information" — not hallucinate.      |

Schema per item (see `src/klerk/eval/golden.py` for the loader):

```jsonc
{
  "id": "Q01",
  "category": "factual",
  "locale": "en",
  "question": "...",
  "expected_answer": "<reference answer>",
  "expected_substrings": ["...", "..."],     // for substring containment
  "expected_doc_ids": ["doc_a", "doc_b"],    // for retrieval recall
  "should_say_dont_know": false,             // trick subset only
  "notes": "optional"
}
```

Items reference the synthetic Fata Organa Solusi corpus generated by
`klerk synth gen` (see `DATA_GENERATION.md` and `src/klerk/synth/specs.py`).

## 2. Metrics

Three layers, each producing a JSON file under `data/output/eval/`:

### 2.1 klerk custom 5-axis rubric (`klerk eval run --rubric`)

| Axis                 | What it measures                                                                  |
|----------------------|-----------------------------------------------------------------------------------|
| `retrieval_recall`   | Fraction of `expected_doc_ids` present in the retrieved chunks.                   |
| `substring_coverage` | Fraction of `expected_substrings` present in the answer text.                     |
| `citation_grounded`  | Fraction of inline `[chunk_id:line]` citations that resolve to retrieved chunks.  |
| `locale_match`       | 1.0 if the answer's locale matches `expected.locale`, else 0.                     |
| `confidence`         | klerk's self-reported confidence on `[0, 1]` from the CRAG judge.                 |

Per-axis scores are averaged into a single `mean` for the headline number.
Aggregates broken down by `category` and `locale`.

### 2.2 RAGAS baseline (`klerk eval run --ragas`)

`ragas>=0.2.0` reference scores for context-precision, context-recall,
faithfulness, and answer-relevancy. Used as a calibration anchor for the
custom rubric. Scores are NOT directly comparable across the two frameworks
(weights differ) — they serve as independent corroboration.

### 2.3 Latency

TTFT (time-to-first-token) and total latency captured per request through
the FastAPI `LatencyMiddleware`. Reported as p50 / p99 aggregates across
the 20-Q run. Surfaced live in the Studio status bar.

## 3. LLM-as-judge — explicit bias disclosure

The `citation_grounded` and `confidence` axes are scored with the same
`nemotron-3-nano-omni` that generated the answers. This is a known
self-grading bias: absolute scores trend ~10-15% higher than a neutral
judge would assign. By-category and by-locale **deltas** are reliable —
the bias is roughly constant. We do not have a second model to use as a
neutral judge (the brief mandates Nemotron only).

Retrieval recall and substring coverage are deterministic — they don't
go through the LLM — so the headline numbers are not bias-contaminated.

## 4. Trick-question handling

Q19 and Q20 don't have grounded answers in the corpus. `should_say_dont_know=true`.
klerk's "I don't know" surface is structural: when the retrieval pipeline
returns no chunks above the rerank threshold, or the CRAG judge marks the
context as ungrounded, `/chat` emits `confidence: 0.0` with an escalation
prompt instead of generating prose. The eval marks the trick question as:
- PASS  if the answer contains "don't" / "don't know" / "no information" /
              Bahasa equivalent, AND `confidence` is below 0.3.
- FAIL  otherwise (a hallucinated answer with any confidence is a fail).

## 5. Run instructions

```bash
# Once the corpus and index exist:
klerk synth gen
klerk index build --src data/synth/fata_organa --rebuild

# Eval:
klerk eval run --ragas --rubric
# → data/output/eval/{rubric,ragas}.json

# Headline numbers + per-Q table are written below by step 11.
```
